# SMC 正統開發方式 — Top-Down 多時間框架策略回測

> 這是 SMC 交易員實際的開發流程的機械化版本:**先用日級別**定結構偏向與
> 關鍵區 (POI), 再下到 60m 等價格回落進區、以 LTF 結構翻轉確認後進場。
> 程式:`smc_mtf.py`。資料:TWII 60m (3年) + GC 60m (2.4年) 交叉檢查,
> 日線由 60m 就地重採樣、只用已完成的日線 (無未來函數)。

---

## 1. 規則 (全機械化)

**HTF 日線層**
- 結構偏向:日線 CHoCH/BOS 判定 trend;**只在日線偏多時做多**。
- 需求區 POI:
  - 日線看漲 FVG 區 `[high[d-2], low[d]]` (SMC_DAILY.md 已驗證最強日線事件)
  - 正統訂單區塊 OB:多頭突破日 (CHoCH/BOS) 前最後一根陰線的 `[low, high]`
- POI 失效:日收盤跌破區塊下緣、或 60 日未被觸及。

**LTF 60m 層**
- 價格回落觸及生效中的日線 POI 且收盤守住區塊 → 武裝。
- 武裝期間 60m 收盤向上突破最近已確認轉折高 (LTF 結構確認) → 次根開盤進場。
- 停損 = POI 下緣 − 0.5×ATR60;出場 = 固定 RR 或 ATR 吊燈 (皆測)。

### 驗證

- 樣本內外 70/30:兩種出場皆內外同號為正 ✓。
- 敏感度:arm_window {15/30/60} 結果不變、日線 pivot L {3/5} 幾乎不變、
  RR {1.5/2/3} 全正 (勝率 75%/63%/64%) — 參數幾乎無關緊要, 結構本身在做事。
- **跨市場是亮點**:在 GC 上單框架 bos+吊燈 幾乎打平 (總/DD 1.11),
  MTF 版 總/DD 3.24 — HTF POI 濾網把 edge 帶到了原本沒有 edge 的市場。

## 3. 結論 — 把整個系列的故事串起來

| 進場哲學 | 驗證結果 |
|---|---|
| 原版:單框架掛限價等回踩 OB | ✗ 全滅 (SMC_STRATEGY.md §7) |
| 單框架順突破 (bos+吊燈) | ✓ TWII 60m 最高總利潤 (PF 4.0) 但跨市場弱 |
| **正統 SMC:日線 POI 回踩 + 60m 結構確認** | ✓ 單筆品質最高 (+731 vs +306)、勝率最高 (63-75%)、**跨市場最穩** |

**「等回踩」本身沒有錯, 錯的是沒有 HTF 背景與 LTF 確認**:
原版在任意結構翻轉後盲目掛限價單, 接到的是下跌中繼;正統 MTF 流程
要求 (1) 日線結構偏多、(2) 回踩的是「日線級別」的需求區、(3) 觸區後
還要等 60m 結構自己翻多才進 — 三道濾網把每筆交易期望從負轉成 +731 點。

**取捨**:MTF 版 3 年只有 ~19 筆 (訊號稀少), 總利潤低於單框架順突破;
但單筆期望 2.4 倍、勝率高、跨市場通用 — 適合搭配大部位/低心理負擔操作。
兩者可並用:順突破當主力、MTF 回踩當加碼點。
