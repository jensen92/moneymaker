# 進出場策略驗證流程 (Strategy Pipeline)

把一個交易構想從「假設」推進到「真實資金進出場」的標準流程。
共 8 道閘門 (G0–G7), **依序通過, 不可跳關**。任一關失敗 → 回爐或否決,
不得帶病闖關。本流程提煉自本 repo 的實戰教訓 (黃金抗過擬合驗證、
小麥 ZW 離群年否決、ATR 天花板弱 base 否決、C vs D EV 對抗等)。

---

## 0. 三條鐵律 (無幻覺紀律)

任何違反以下鐵律的報告一律無效, 必須重做:

1. **數字必須來自本次執行**。報告中每一個績效數字 (勝率/PF/MaxDD/EV/年報酬...)
   都必須來自本次 session 實際跑出的指令輸出, 且報告需附「指令 + 原始輸出節錄」。
   禁止憑記憶引用、禁止直接抄舊文件的數字 (舊文件可能因資料更新/程式修改而過期;
   要引用就重跑)。
2. **事實、推論、假設三分**。報告中每個論斷標明類別:
   - 【事實】= 有本次執行輸出佐證;
   - 【推論】= 由事實推導, 寫明推導鏈;
   - 【假設】= 未驗證, 必須列入待驗證清單, 不得作為過關依據。
   樣本內調出來的門檻一律標 ⚠️樣本內 (例: P/C≥138 這類門檻)。
3. **單一程式碼真相**。回測、模擬、實單訊號必須重用同一份訊號/出場邏輯
   (import 同一模組), 禁止為模擬或實單另寫一份「差不多」的邏輯 —
   本 repo 的幽靈平倉、出場月誤報等 bug 全部源自邏輯分叉。

## 1. 流程總覽

| 閘門 | 名稱 | 產出 | 過關才能做什麼 |
|---|---|---|---|
| G0 | 假設登記 | 登記文件 | 開始寫程式 |
| G1 | 資料完整性 | 資料檢核輸出 | 開始回測 |
| G2 | 回測正確性 | 基準回測結果 | 開始優化 |
| G3 | 抗過擬合六檢 | 六檢結果表 | 進入統計檢定 |
| G4 | 統計檢定 | 顯著性結論 | 凍結參數 |
| G5 | 凍結模擬 | 模擬帳對帳單 | 申請實單 |
| G6 | 實單閘門 | 實單 SOP + kill-switch | 小部位實單 |
| G7 | 上線監控 | 定期健檢報告 | 維持/加碼/退役 |

門檻數字為預設值, 可依商品特性調整, 但**調整必須在 G0 登記時寫明理由**,
不得事後放寬來讓策略過關。

## 2. G0 — 假設登記 (pre-registration)

在跑任何回測**之前**, 先寫下並存檔 (研究 md 或程式 docstring):

1. **經濟直覺**:這個 edge 為什麼存在?誰在把錢輸給你 (行為偏誤/結構性資金流/
   風險溢酬)?說不出來的 edge 大概率是雜訊。
2. **可否證的預測**:例如「黃豆因收割季供給壓力, 10 月進場持有至 5 月,
   多數年份應為正報酬」。要能被資料否證。
3. **預先宣告** (之後不得偷改):
   - 市場、頻率、方向 (多/空/雙向)
   - 進場條件、出場條件 (停損/停利/時間出場) 的完整規格
   - 要掃描的參數與範圍 (掃描組數 N 記下來, G4 要用)
   - 主要指標 (預設: 報酬/回撤比 + 逐年中位數) 與過關門檻
   - 樣本切分方式 (IS/OOS 切點日期)
4. **禁改條款**:OOS 資料在 G3 之前不得用於任何調參。

> 為什麼:先看結果再定假設 = 資料窺探。archive/STRATEGY_AUDIT.md 缺點 4
> 記錄過教訓 — 對同一份資料反覆掃 108 組參數而無樣本外紀律。

## 3. G1 — 資料完整性

跑回測前逐項檢核並在報告列出輸出:

- [ ] **資料區間與根數**:印出起訖日期、K 棒/交易日數量。區間太短直接否決
      (日線策略至少 8–10 年, 需涵蓋至少一次空頭與一次盤整)。
- [ ] **除權息**:台股用 `data_adj/` (還原價) 跑訊號與績效;若只有未還原價,
      報告必須註明訊號誤差風險。
- [ ] **生存者偏差**:股票池是否含已下市股?若否, 報告註明高估風險。
- [ ] **空值/零值**:停牌/零收盤價要跳過 (曾有除以零崩潰 bug);
      API 回空資料不得覆寫既有歷史 (曾有空覆寫 bug)。
- [ ] **未來函數快篩**:訊號當根只能用「當根收盤及之前」的資訊;
      任何 rolling 計算確認無 center/shift 錯誤;進場價必須是訊號之後
      才能成交的價格 (本 repo 慣例: 訊號日收盤確認 → 次一交易日開盤進場)。
- [ ] **時區/交易時段**:期貨小時線確認 session 邊界正確。

## 4. G2 — 回測正確性

- **成本從嚴**:台股 = 手續費 0.1425% 雙邊 + 證交稅 0.3% + 滑價 0.1% 單邊;
  期貨 = 各商品每邊成本 + 滑價。零成本回測結果一律無效。
- **執行假設**:次日開盤進場、漲停跳過、整股單位、流動性門檻
  (股價 > 10 元、日成交值 > 5 千萬 之類) 明列於報告。
- **部位規則**:單筆風險 = 權益 1%、總名目曝險 ≤ 權益 × 槓桿上限。
  本 repo 實證 1x 整體優於 2x (同步停損期在槓桿下放大), 預設 1x。
- **基準對照**:同期 buy & hold / 大盤 / 現有生產策略 (如 D) 至少列一個。
  打不贏「什麼都不做」或現有策略的新策略, 直接在此止步。
- 優先重用現有引擎: `backtest.py` (台股組合)、`futures_backtest.py`、
  `gold_daily_backtest.py`、`ma_backtest.py` 等, 不要重造輪子。

---

## 5. G3 — 抗過擬合六檢 (核心關卡)

六項全查, 逐項在報告給 ✅/❌ 與證據。**任何一檢不過 = 不得進 G4**。

### 3.1 樣本內/外對切 (IS/OOS)
- 只在 IS 調參;凍結後跑 OOS。OOS 必須獨立獲利 (PF ≥ 1.1 且總損益 > 0)。
- OOS 只能看一次。看完不滿意回去改參數 = OOS 已污染, 需換切點或等新資料。

### 3.2 Walk-forward
- 滾動窗重新優化 → 各窗 OOS 串接的權益曲線必須整體獲利,
  且獲利不集中於單一窗。

### 3.3 參數平台 (plateau, 非尖峰)
- 最優參數的鄰格 (每個參數 ±1~2 步) 必須全數正期望,
  且鄰格績效衰退 ≤ 50%。
- **單點尖峰 = 直接否決** (黃金 MA150 教訓:小時線單點尖峰、鄰居不確認,
  最後靠日線 26 年跨 regime 複驗才定案)。

### 3.4 逐年分解
- 逐年報酬表:中位年報酬 > 0、正報酬年占比 ≥ 55%。
- **移除最佳單一年後總損益仍須 > 0** (小麥 ZW 教訓:勝率 38%、中位 −3.4%,
  正報酬全靠 2021 單一離群年 → 否決移除)。
- 工具參考: `season_validate.py` 的逐年 % 重放法。

### 3.5 跨 regime / 跨頻率複驗
- 換一個頻率 (小時→日線) 或拉長歷史 (含空頭、盤整年) 套同一概念,
  方向性結論必須一致 (黃金趨勢閘的「決定性第 4 關」做法,
  見 `gold_gate_validate.py`)。

### 3.6 對抗複驗 (adversarial re-check)
- 主動當自己的反方, 至少回答:
  - 增益是絕對的, 還是「對弱 base 的相對改善」? 換強 base 還在嗎?
    (ATR 天花板教訓:增益只是修飾弱 base → 否決)
  - 和現有策略的報酬流相關性多高? 相關 > 0.6 的「新」策略是假分散
    (動能家族 K/O 月報酬相關 0.95 的教訓)。
  - 獲利是否過度依賴 <10% 的右尾大贏家?若是, 報告必須寫明脆弱性。

## 6. G4 — 統計檢定 (區分 edge 與運氣)

- **最少樣本**:全樣本 ≥ 60 筆交易、每個切分 ≥ 30 筆。
  季節策略例外 (每年一筆):≥ 15 年、勝率 ≥ 55%、中位 > 0。
  樣本不足 → 判定只能是「觀察」, 不得上實單。
- **Bootstrap**:對單筆報酬 (R 或 %) 重抽 ≥ 5000 次,
  平均值 95% CI 下界 > 0 才算顯著。
- **多重測試折減**:G0 登記的掃描組數 N 必須列在報告。
  t-stat = 平均單筆報酬 ÷ 標準差 × √樣本數;N > 20 時門檻從 2 提高到 3
  (掃愈多, 最佳組合愈可能是運氣;學術界對因子研究的建議即 t > 3)。
- **誠實解讀**:「差異不顯著」是合法結論 — C vs D 的 EV 對抗即是如此,
  最後決策是並用分散而非硬選一個。不要為了給結論而捏顯著性。

---

## 7. G5 — 凍結模擬 (paper trading)

- **凍結**:參數與程式版本凍結 (記下 commit hash)。模擬期間發現想調參 →
  回 G0 重跑全流程, 模擬計時歸零。已驗證穩健的策略標註「勿再調參」。
- **凍結帳本**:用 `sim_log.py` 模式 — 已平倉交易逐日累加寫入,
  以唯一鍵去重, **絕不因事後程式/資料變動而改寫舊紀錄**。
  即時重算的「回測式對帳」不算模擬證據。
- **最低時長**:≥ 20 筆已平倉交易 或 ≥ 40 個交易日 (取較長者)。
  季節策略至少完整模擬一個季節窗。
- **對帳三項**:
  1. 訊號一致:模擬訊號與回測引擎在同日輸出一致 (單一程式碼真相);
  2. 滑價:實際可成交價 vs 回測假設, 平均滑價 ≤ 假設 × 2;
  3. 分布:模擬期 R 分布落在回測 bootstrap 90% 區間內
     (落外不必然否決, 但必須解釋)。

## 8. G6 — 實單閘門 (真實資金)

G0–G5 全過才進此關。過關不代表保證獲利, 只代表「尚未被否證」;
以下規則是虧損時的最終防線, **上線前寫死, 虧損中不得放寬**:

### 起步規模
- 首 20 筆 (或首 3 個月) 部位 ≤ 目標規模 1/4, 或交易所最小單位。
- 通過起步期 (績效落在回測分布內) 才逐步加到目標規模。

### Kill-switch (任一觸發 → 全部平倉、停止進場、重跑 G3 健檢)
- 實單回撤 > 回測 MaxDD × 1.5;
- 連續虧損筆數 > 回測最長連虧 + 3;
- 實際滑價連續 10 筆 > 假設 × 2;
- 資料源異常 (API 空回應、價格跳變) 未排除前。

### 每日實單 SOP
1. 收盤後跑訊號腳本 (報告中列出確切指令), 核對程式版本 = 凍結 hash;
2. 訊號成立 → 次一交易日開盤進場, **成交後立即掛停損單**;
3. 部位大小用公式算 (單筆風險 1% 權益), 不憑感覺;
4. 成交價/滑價記入實單 ledger (與模擬帳同格式);
5. 漲停跳過、流動性不足跳過 — 跳過也要記錄, 供追蹤訊號覆蓋率;
6. **持倉中禁止手動干預**:提前出場/移停損/加碼皆屬改策略, 走 G0。

---

## 9. G7 — 上線監控與退場

- **滾動健檢**:每 20 筆實單交易 (或每季) 對照回測分布:
  滾動平均 R、勝率、滑價。連續兩期落在 bootstrap 5% 尾端 → 降半部位並複查。
- **年度複驗**:每年用含新資料的全樣本重跑 G3 六檢 (不重新優化, 只驗證)。
- **退役條件**:年度複驗任一檢轉 ❌、或 edge 的經濟直覺已消失
  (結構改變, 例如稅制/交易制度變更) → 降部位至零, 移入 archive 並寫結案紀錄。
- 任何參數修改 = 新策略, 從 G0 重來。

## 10. 判定矩陣

| 狀態 | 條件 | 動作 |
|---|---|---|
| **生產** | G0–G5 全過 + 起步期通過 | 目標規模實單, G7 監控 |
| **觀察** | G3 全過但 G4 樣本不足 / G5 未滿 | 只模擬, 累積樣本 |
| **回爐** | G3 個別檢未過但概念仍有直覺支撐 | 改規格回 G0, OOS 換切點 |
| **否決** | 尖峰參數 / 靠單一離群年 / 弱 base 相對增益 / OOS 虧損 | 寫入結論封存, 免重跑 |

否決結論寫入 `PROJECT_STATE.md` §4 研究結論封存, 防止未來重複試錯。

## 12. 產出:驗證報告

每次完整跑完流程, 依 `references/report-template.md` 產出一份驗證報告
(命名 `<策略名>_VALIDATION.md`), 必含:

- 逐閘門 ✅/❌ 與證據 (指令 + 輸出節錄);
- G0 登記內容原文 (證明未事後改假設);
- 掃描組數 N 與統計檢定結果;
- 最終判定 (生產/觀察/回爐/否決) 與下一步;
- 若判定「生產」:完整實單 SOP、部位公式、kill-switch 數值。

> 最後提醒:通過所有閘門只代表策略「在歷史資料上未被否證」。
> 市場會變, 回測不保證未來 — kill-switch 與部位控制才是真實資金的
> 最終保險, 這兩項的優先級永遠高於任何回測數字。
