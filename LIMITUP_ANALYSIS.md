# 漲停前特徵統計與篩選 — 方法、結論與 vs 現行 K/D 策略差異

> 對應程式:`limitup_study.py`。本文說明「隔日漲停 (+10%) 前一日長什麼樣子」的
> 量化方法、如何組成篩選, 以及它和目前生產策略 (K/D, Minervini VCP 突破變體)
> 在**問題本質上的結構差異**。實證數字需在備妥日線資料 (`download_all.py`) 後,
> 執行 `limitup_study.py` 產生 (見文末「如何跑」)。

---

## 1. 為什麼這是「不同的問題」

現行 K/D 的目標不是抓漲停,而是**騎乘數週的主升段**:

- 進場點 = 突破日本身 (收盤突破 20 日高 + 量 ≥ 2 倍 50 日均量),
- 進場當日刻意把**漲幅上限壓在 6%** (`gain_cap`),*緊貼樞軸點、絕不追高*,
- 且回測引擎在**漲停日直接跳過不進場** (`PROJECT_STATE` §2「漲停跳過」),
- 出場靠 MA50 移動停損 / +20% 賣半 / 衝頂,持有數週~數月。

也就是說,現行策略**刻意迴避**「今天噴出/漲停」這種當下爆發,它要的是
「安靜突破後的持續走勢」。而本次任務問的是相反的事:

> **在一檔股票『隔天就漲停』之前,前一天收盤時有哪些可觀察的前兆?**

這是**單日、事件式**的預測 (horizon = 1 日),與 K/D 的**多週趨勢騎乘**是
兩種不同的獵物。所以「差異」不只是參數不同,而是**目標函數不同**。

## 2. 方法 (`limitup_study.py`)

**事件定義**:台股日漲跌幅上限 10%,因跳動單位取整,以
`當日收盤 / 昨收 − 1 ≥ 9.5%` 判定漲停 (`LIMIT_THRESH=0.095`)。
`--sealed` 可再要求收在當日最高附近 (封板鎖死),更嚴格。

**樣本**:對「可交易宇宙」(收盤 ≥ 10、日成交值 ≥ 5 千萬、MA200 有效 —— 與
`_d_features` 同口徑,確保和 K/D 的比較公平) 的**每一個股票日 T**,只用
「到 T 收盤為止」的資訊抽一組前兆特徵,標記 `label = T+1 是否漲停`。

**前兆特徵** (皆 T 收盤可得,涵蓋六個正交維度):

| 維度 | 特徵 | 直覺 |
|---|---|---|
| 相對強度 | `rs_rank` (126日報酬百分位) | 資金點火偏好領漲股 |
| 位置 | `prox_52wh`=收/52週高、`near20h`=收/20日高 | 貼近前高 → 突破就緒 |
| 動能 | `ret5/ret20/ret60`、`upstreak` | 多框架動能與連漲 |
| 趨勢 | `trend_score` (0~6 模板得分) | 是否多頭排列 |
| 壓縮 | `contraction`=ATR5/ATR14、`day_range` | 爆發前波動/振幅收斂 |
| 量能 | `vol_dryup`(昨量<均量)、`vr5`(近5日均量/50日均量) | 量縮蓄勢 / 量能翻升 |
| 其他 | `rsi14`、`ext20`(乖離)、`gain_today` | 過熱/延伸排除 |

**輸出四塊**:

1. **前兆日 vs 全體的特徵均值** — 漲停前一日相對全市場偏高/偏低在哪。
2. **各特徵十分位的『隔日漲停條件機率』與 lift** — 找出**單調前兆**
   (某特徵越高、隔日漲停率越高 → lift>1)。這是「哪些特徵真的先於漲停」的核心。
3. **`LU_SCREEN` 篩選評估** — 把最強前兆組成一條**可解釋的固定規則**,報
   precision (候選中隔日真漲停比例) / recall (全體漲停被覆蓋比例) / 每日候選數,
   並與單維最強前兆對照。刻意用可讀規則而非過擬合權重,方便與 K/D 逐條對照。
4. **與 K/D 對比** (見 §4)。

`LU_SCREEN` 規則 (v2,已依 122 萬股票日實證校準,取各特徵 lift 明顯 >1 的分位起點):

```
RS ≥ 0.90  且  近20日漲幅 ≥ 15%  且  近5日均量/50日均量 ≥ 1.5 (量增)
且  貼52週高 ≥ 0.90  且  當日紅K (gain_today > 0)
```

> ⚠️ **v1(壓縮蓄勢版)已被實證否決** —— 見 §3。原規則要求波動/振幅壓縮＋量縮,
> 但真實資料顯示那些方向與隔日漲停**負相關**,v1 precision 1.04% < 基準 2.08%(lift 0.5x)。
> v2 改用『同向動能』因子。

此外提供 **`LU_SCORE` 綜合分數**(`build_lu_score`):把五個正交前兆
(`rs_rank / ret20 / vr5 / prox_52wh / gain_today`)各自的全樣本百分位平均成 0~1 分數。
比固定 AND 門檻更適合『排名取前 N』—— 用分數高低即可平滑調 precision/recall 取捨,
不必猜死門檻。腳本會印出前 0.1% / 0.5% / 1% / 2% / 5% / 10% 各切點的
precision / recall / lift,形成一條可挑選工作點的取捨曲線。

## 3. 實證前兆型態 (2026-07,`data_adj` 除權息還原 + 封板鎖死)

樣本:**1,218,418** 個股票日,**25,377** 個封板漲停事件,基準隔日漲停率 **2.083%**。

**核心結論:隔日漲停的前兆是「動能點火/延伸」,不是「安靜壓縮」。**
差異倍率 (漲停前一日均值 / 全體均值) 最大的全是動能/擴張類:

| 特徵 | 差異倍率 | 最高分位 lift | 方向 |
|---|---|---|---|
| `gain_today` 當日漲幅 | **6.77x** | 3.72x | 前一日常已收中大紅 (點火) |
| `ret5` 近5日 | 4.14x | 3.67x | 短期強動能 |
| `ext20` 乖離 (收/MA20) | 3.73x | 3.88x | **越延伸越易漲停** (與 v1 假設相反) |
| `ret20` / `ret60` | 3.45x / 3.07x | 3.65x / 3.48x | 中長期強動能 |
| `day_range` 當日振幅 | 1.65x | 3.67x | **振幅越寬越易漲停** (非壓縮) |
| `rsi14` | 1.13x | 2.71x | 高 RSI (強勢, 非超賣) |
| `rs_rank` 相對強度 | 1.24x | **2.70x** (單調最漂亮) | 領漲股 |
| `vr5` 量比 | 1.46x | 2.52x | 量能已擴張 |
| `upstreak` 連漲天數 | 1.56x | 2.06x | 連續走強 |
| `contraction` ATR5/ATR14 | **1.11x** | 2.07x | **波動擴張** (與 v1 假設相反) |
| `vol_dryup` 昨日量縮 | **0.80x** | ~1.0 (無訊號) | **量縮反而更少見** |
| `prox_52wh` 貼52週高 | 1.02x | 2.16x (僅極端分位) | 至高點才有效, 均值訊號弱 |
| `near20h` 貼20日高 | 0.99x | 2.27x (U 形) | 貼平/遠離兩端才高 |

**被否決的 v1 假設**:壓縮 (`contraction`↓/`day_range`↓) 與量縮 (`vol_dryup`) 原以為是
「彈簧蓄勢」,實測方向全反 —— 對照篩選「波動壓縮」precision 僅 0.69% (lift **0.3x**)。
「彈簧越緊彈越猛」對『預測明天這一根』不成立;會漲停的票明天,今天多半**已經在噴**。

**最強單一前兆是相對強度**:`RS ≥ 0.90` 單條即 precision 4.37% / lift 2.1x / recall 49%;
`貼52週高 ≥ 0.95` 為 3.57% / 1.7x。這也是 v2 `LU_SCREEN` 以 RS + 動能為骨幹的原因。

## 4. 與現行 K/D 篩選的差異 (核心)

| 面向 | 現行 K / D (VCP 突破) | 漲停前兆篩選 LU_SCREEN |
|---|---|---|
| **預測標的** | 未來數週的**主升段趨勢** | **隔日單日**是否漲停 |
| **進場時點** | 突破日**當天**收盤確認後買 | 前兆日收盤篩、隔日開盤搶 |
| **對「爆發」的態度** | **迴避**:`gain_cap` 6%、漲停日跳過、不追高 | **鎖定**:就是要找即將爆發的那根 |
| **量能條件** | 突破日**爆量** ≥ 2× 均量 (需求確認) | 前兆日量能**已擴張** (`vr5` 高分位 lift 2.5x) |
| **持有** | 數週~數月,MA50 移動停損 | 事件型,極短 (次日/數日) |
| **主要因子** | 突破 + 爆量 + 緊貼樞軸 + 高 RS + 52週高接近度 | 高 RS + 強動能 + 延伸 + 量增 + 貼前高 |
| **樣本/頻率** | 全期 ~560–580 筆 (高品質稀少) | 視門檻,每日候選數不同 |
| **評估指標** | PF / MaxDD / 勝率 / MAR (部位級) | precision / recall / lift (事件級) |

**關鍵洞察 (實證)**:兩者共享「高 RS」,但在**進場位置**上**方向相反** —— K/D 要
**緊貼樞軸、gain ≤ 6%、不追高**,LU_SCREEN 找的卻是**今天已經在噴、延伸、量增**的票。
`gain_cap = 6%` 這個旋鈕直接排除了 LU_SCREEN 想抓的「即將續噴的大陽線」。三個量化指標
把差異釘死 (2026-07 實測,基準隔日漲停率 2.083%):

| 指標 | 策略 K | 策略 D |
|---|---|---|
| 訊號數 | 829 | 2,264 |
| 訊號隔日自身漲停率 | **1.81%** (低於基準) | 2.47% (微高於基準) |
| 全體漲停被『事前一日』覆蓋 | 14/25377 = **0.1%** | 54/25377 = **0.2%** |
| 與 LU_SCREEN 候選 Jaccard | 0.4% | — |

- **K 訊號隔日漲停率 1.81% < 基準** —— 完全符合 K 的設計:`gain_cap 6%` 就是不買會噴的那根。
- **K/D 對全體漲停的事前覆蓋 ~0.1–0.2%** —— 幾乎為零,證明 K/D 幾乎不站在漲停前一日。
- **Jaccard < 1%** —— 兩套候選幾乎不重疊。

> **結論**:LU_SCREEN **不是** K/D 的替代品,而是**完全不同的獵物**(隔日爆發 vs 數週趨勢),
> 兩者近乎正交。**警示**:v2 篩的是『已延伸/動能延續』的票,追高風險高,較適合當
> **動能延續/追強觀察名單**,而**不**應併入 K/D 的部位邏輯 —— 以免破壞 K/D 已驗證的風控。
> 若要當可交易策略,需另評估滑價、封板無法成交、以及買在 climax 的回落風險。

## 5. 如何跑

```bash
python3 download_all.py                    # 先備妥日線 (data/ 或 data_adj/)
python3 limitup_study.py                   # 用 data/ (未還原除權息)
MM_DATA_DIR=data_adj python3 limitup_study.py --sealed   # 除權息還原 + 只計封板
python3 limitup_study.py --min-turnover 100e6            # 收緊流動性門檻
MM_DATA_DIR=data_adj python3 limitup_study.py --sealed --dump-md   # 另存 markdown 報告
```

輸出:stdout 完整報告 (前兆均值表 / 十分位 lift / LU_SCREEN 評估 / K-D 對比),
另存 `limitup_lift.csv` (各特徵十分位 lift 明細,已 gitignore)。加
`--dump-md [PATH]` 會把完整四段結果另存成一份 markdown 報告 (預設
`limitup_report.md`,已 gitignore),可直接貼上或分享,不必手動整理終端輸出。

**注意**:Yahoo 原始價未還原除權息,除權息日會產生假跌 (不會誤判漲停,但會影響
動能特徵);要精確請用 `data_adj/`。漲停/跌停的**成交流動性**本研究以成交值近似,
未模擬「排隊掛不到」的實際成交難度 —— 事件研究看的是**前兆的統計顯著性**,
真要當可交易策略需另外評估滑價與封板無法成交的問題。
